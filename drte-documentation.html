Installation & Permissions

Step 1: Prerequisites & Install

Before installing DRTE, ensure your Salesforce org meets the following requirements:

  • Salesforce Enterprise, Unlimited, or Performance Edition
  • Administrator access to install managed packages
  • Reach out to info@codeylabs.com for latest GA installation url

Step 2: Configure User Permissions

Assign the appropriate permission sets (DRTE User, DRTE Admin, DRTE ESign Guest User) to relevant users based on their role.

Watch Permissions Setup Video


Core Admin Configuration

1. Define Allowed Objects

A crucial security step: Whitelist the Salesforce objects that can be queried for dynamic document generation using the Custom Metadata Type.

  • Navigate to Custom Metadata Types.
  • Update the 'Allowed Objects' to add records using the exact API Names (e.g., `Opportunity`, `Account`).

2. Deploy & Configure LWC

Make the DRTE component live on your target object's record page for users to access the documents.

  • Use **Lightning App Builder** to place the LWC.
  • Set component to **Read Only**.
  • Define the **Default Template Record ID**.

3. Activate E-Signature UI

Enable the final user interface for tracking and initiating signature requests on the record page.

  • Check **'Enable E Signature Request feature'** on the DRTE component.
  • Drag-and-drop the **E Signature Request list** component (enable hierarchy view).


Document Template Design

Template Foundation & Header

Learn how to create the base template record and securely embed your company logo in the document header.

Key Configuration Steps:

  • Create the Doc Template record and **Deactivate** it (for default loading).
  • Upload Logo to Salesforce **Files**, make it **Public**, and get the **Direct Image URL**.
  • Create a **Paragraph (Static)** section for the header content.

Dynamic Data Sections

Configure the Vendor/Customer information and the dynamic line item tables using SOQL queries.

Terms & E-Signature Placeholders

Finalize the document with legally required text and the signature fields.

Final Template Steps:

E-Signature Configuration

E-Signature Portal & Security

Setup the public-facing Salesforce Digital Experience Site for external signers to access documents securely.

Key Security Steps:

  • Create **'Build Your Own (Aura)'** site (e.g., ESign) with Public Access.
  • Assign the **'DRTE ESign Guest User'** Permission Set.
  • Set up **Sharing Rules** (Read Only) for E Signature Request/Logs objects.

Automation & Branding

Finalize the workflow by setting the automation and branding details in DRTE Custom Settings.

Custom Settings:

  • Set the **ESign Guest Site URL**.
  • Define **Org Wide Email** and **Email Template IDs**.
  • Configure **Company Header Image URL** for email branding.
  • Add your Salesforce domain to **Trusted URLs** for security.

Troubleshooting & FAQs

Check that the template ID is correctly configured and that the user has proper permissions. Ensure the SOQL query syntax is correct and returns data. Watch our troubleshooting video for step-by-step guidance.

Verify that field API names are spelled correctly and that the fields exist on the object. Check field-level security to ensure users can access the fields. Use the debug mode to identify specific field issues.

Ensure that signature fields are properly placed on the document. Check that the signing workflow is active and configured correctly. Verify that email notifications are being sent and not blocked by spam filters.

Check the Salesforce Recycle Bin within 15 days of deletion. If the document was attached to a record, check the record's Files or Attachments. Consider implementing a backup strategy for critical documents.

Check email deliverability settings in Salesforce. Verify that sender email addresses are authorized. Check that recipients aren't blocking emails. Review email logs for delivery errors.

Verify that all images and assets are accessible. Check for special characters that might cause encoding issues. Ensure templates don't exceed size limits. Test with simplified versions to isolate problems.
